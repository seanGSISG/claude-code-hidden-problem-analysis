# 데이터 소스와 라벨링

> **이 문서를 작성한 이유.** 이 저장소의 이전 개정판은 데이터셋을 "ubuntu-1, 단일 머신, 1,735개 JSONL 파일"로 기술하였습니다. 4월 16일의 재감사 결과, 실제 데이터 자산은 **세 개의 라벨링된 환경**에 걸쳐 있으며, 이전에 발표된 수치들은 그중 가장 큰 환경의 스냅샷 부분집합에 해당한다는 사실을 확인하였습니다. 이 문서는 전체 소스 집합과 교차 분석에 사용되는 라벨, 그리고 현재 수치와 과거 스냅샷 간의 관계를 선언합니다.

## 1. 환경 라벨

모든 데이터셋은 다음 차원으로 태깅됩니다. 하위 분석은 이 라벨을 명시적으로 필터링할 것이 요구됩니다.

| 필드 | 값 |
|---|---|
| `machine` | `zbook` \| `win-1` \| (향후: 기타 랩 머신) |
| `account_tier` | `max20` \| `max5` \| (향후: `pro`, `api`) |
| `cc_mode` | `stock`(네이티브 `~/.claude`) \| `override`(GrowthBook 플래그 오버라이드가 활성화된 분리 오버라이드 환경; 추가 구성요소는 비공개로 유지) |
| `cc_version` | 예: `v2.1.91`, `v2.1.109`, `v2.1.110` — 대표 버전이며, 실제 값은 데이터셋 기간에 따라 달라집니다 |
| `proxy_stack` | **공개** 측에 활성화된 프록시/계측 계층의 배열(예: `cc-relay`, `claude-code-cache-fix`). 비공개 구성요소는 내부적으로 기록하되 공개하지 않습니다. |

## 2. 현재 데이터셋 (2026-04-22 기준)

| dataset | machine | tier | mode | version | proxy_stack | JSONL 파일 | messages | proxy 요청 | 크기 |
|---|---|---|---|---|---|---|---|---|---|
| `ubuntu-1-stock` | zbook | max20 | stock | v2.1.91 | cc-relay | 2,098 | 246,954 | 45,884 (cc-relay) | 911 MB |
| `ubuntu-1-override` | zbook | max20 | override | v2.1.109 | (비공개 운영 도구 — 공개하지 않음) | 2,324 | 263,630 | (후속 작업) | 948 MB |
| `win-1-stock` | win-1 | max5 | stock | v2.1.110 | 투명 프록시 + `claude-code-cache-fix` 1.10.0 (공개) | 171 | 1,565 | (후속 작업) | 5.7 MB |

**총합: JSONL 파일 4,593개 / 메시지 512,149개 / proxy 요청 45,884건 / 약 1.9 GB**

### 2.1 데이터셋별 비고

- **`ubuntu-1-stock`** — 과거 기준선. 이 저장소의 이전 개정판에 등장하는 모든 수치는 이 데이터셋에서 비롯되며, 기간별로 필터링된 것입니다(proxy는 4월 1–15일, JSONL 일괄 스캔은 선택된 세션 부분집합).
- **`ubuntu-1-override`** — 동일 머신·동일 계정이되, 4월 10일부터 GrowthBook 플래그 오버라이드가 활성화된 분리 오버라이드 환경입니다([01_BUGS.md](01_BUGS.md#growthbook-flag-override--controlled-elimination-test-april-1014) 참조). 이 환경에는 운영 목적상 내부적으로 추적하는 추가 비공개 구성요소가 포함되어 있으나, 이 저장소의 범위에는 포함되지 않습니다. B4/B5 제거 테스트에 사용되었습니다.
- **`win-1-stock`** — 별도의 Max 5x 계정을 사용하는 별개의 Windows 11 머신입니다. 연구 및 검증(버전 지문 확인, 프록시 호환성, [`claude-code-cache-fix`](https://github.com/cnighswonger/claude-code-cache-fix) 등 공개 도구의 독립 테스트)에 사용됩니다. Max 20x ubuntu-1 분석을 통제된 상태로 유지하기 위해 이 저장소의 주요 공개 본문에서는 **의도적으로 제외**됩니다.

### 2.2 공유 저장소 주의사항

ubuntu-1의 `~/.claude/projects/`(stock)과 분리된 오버라이드 환경(override env) 모두 **2026-02-08**까지 거슬러 올라가는 JSONL 항목을 포함하고 있습니다. 이는 두 디렉토리가 과거에는 공유 기록에서 분기되었기 때문이며, 초기 항목들이 양쪽에 모두 존재합니다. 환경 효과 분석(예: 오버라이드 전/후 비교)을 수행할 때는 두 환경이 의미 있게 분기되는 기간을 분리하기 위해 `ts >= '2026-04-10'` 필터를 적용하십시오.

## 3. 이전에 발표된 수치와의 조정

| 발표된 수치 | 해당하는 데이터 | 현재 등가 수치 |
|---|---|---|
| "1,735개 JSONL 파일, 1.0 GB, 단일 머신" ([`03_JSONL-ANALYSIS.md`](03_JSONL-ANALYSIS.md)) | 해당 섹션 작성 시점의 `ubuntu-1-stock` 스냅샷 | 현재 `ubuntu-1-stock`에서 2,098개 파일 / 911 MB (계속 증가 중) |
| "4월 1–15일 35,554건 요청, 251개 세션" ([`13_PROXY-DATA.md`](13_PROXY-DATA.md)) | `ubuntu-1-stock` cc-relay proxy, 4월 1–15일 구간 | `ubuntu-1-stock` 하에서 **45,884건 요청 / 320개 세션 / 4월 1–22일**로 확장 (cc-relay 소스) |
| "532개 JSONL 파일, 158.3 MB (4월 1–8일)" ([`06`, `13 §12`]) | `ubuntu-1-stock` 일괄 스캔 스냅샷 | 과거 데이터이며 재실행하지 않음. 현재 `ubuntu-1-stock` 코퍼스는 더 크지만(2,098개 파일), 당시 계산된 지표에 대해서는 532개 파일 스캔이 권위 있는 자료로 유지됩니다 |
| "오버라이드 이후 4,919건 요청, B4/B5 0건" ([`01_BUGS.md`](01_BUGS.md)) | `ubuntu-1-stock` cc-relay proxy, 4월 11–14일 오버라이드 윈도우 | 과거 수치는 변하지 않음. 오버라이드는 계속되며, 오버라이드 측정과 관련된 4월 14일 이후 데이터는 `ubuntu-1-override`에 대해 추적됩니다 |

**과거 스냅샷 수치는 철회되지 않습니다** — 각 분석이 작성된 시점의 데이터 상태를 기술한 것이며, 원본 세션에서 재현 가능합니다. 여기서 수정되는 것은 라벨링입니다. 머신/환경 기술자인 "단일 머신"은 부정확한 표현이었으며, 이제 명시적인 `ubuntu-1-stock` 라벨로 대체됩니다.

## 4. 저장소 레이아웃

4월 16일부터 위 세 개의 데이터셋은 내부 Postgres 데이터베이스에도 인덱싱되어 있습니다. 다섯 개의 테이블이 원본 파일을 디스크 상에 그대로 유지하면서, 메타데이터·턴 단위 파싱 메시지·proxy 이벤트·플래그 스냅샷을 인덱싱하여 SQL 수준의 교차 분석을 가능하게 합니다. ETL 스크립트는 idempotent하게 설계되어 있으므로 데이터셋 재적재가 안전합니다.
