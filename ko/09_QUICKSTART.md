# 빠른 시작 — 설정 및 자가 진단

> 토큰(Claude가 한 번에 처리하는 텍스트 단위) 소모를 최소화하기 위한 실용적인 단계입니다. 여기 나오는 권장 사항의 기술적 분석이 궁금하시다면 [01_BUGS.md](01_BUGS.md)를 참조하십시오.

---

## 설치

### 처음 설치하는 경우 (권장: npm 방식)

npm은 Node.js(자바스크립트 실행 환경)와 함께 제공되는 패키지 관리 도구입니다. 이 방식으로 설치하면 알려진 캐시 버그를 피할 수 있습니다.

```bash
# 1. npm으로 설치 (Sentinel 버그 없음)
npm install -g @anthropic-ai/claude-code

# 2. 자동 업데이트 비활성화
cat > ~/.claude/settings.json << 'EOF'
{
  "env": {
    "DISABLE_AUTOUPDATER": "1"
  }
}
EOF

# 3. 확인
claude --version   # 2.1.91 이상이어야 함
file $(which claude)   # cli.js로의 심볼릭 링크여야 함
```

위 명령어를 터미널(맥은 Terminal, 윈도우는 PowerShell 또는 명령 프롬프트)에 한 줄씩 복사해서 붙여넣으면 됩니다. 1번은 Claude Code를 설치하고, 2번은 자동 업데이트를 꺼주고, 3번은 잘 설치됐는지 확인하는 것입니다.

### 이미 Standalone(독립 실행 파일)으로 쓰고 있는 경우

```bash
# 1. v2.1.91로 업데이트
claude update

# 2. 자동 업데이트 비활성화 (기존 settings.json에 추가)
# "env" 섹션에 "DISABLE_AUTOUPDATER": "1" 추가

# 3. 확인
claude --version   # 2.1.91 이상이어야 함
```

마찬가지로 터미널에 복사해서 붙여넣으면 됩니다. 2번은 기존 설정 파일을 직접 열어서 수정해야 합니다. `~/.claude/settings.json` 파일을 텍스트 편집기로 열고, `"env"` 안에 `"DISABLE_AUTOUPDATER": "1"`을 추가합니다. 자동 업데이트를 끄는 이유는, 나중에 나올 버전에서 새로운 문제가 생길 수 있기 때문입니다.

## 피해야 할 동작

아래 동작들은 토큰을 불필요하게 많이 소모합니다. 토큰을 많이 쓸수록 사용량 한도에 빨리 도달합니다.

| 동작 | 이유 | 영향 |
|------|------|------|
| `--resume` | 전체 대화 히스토리를 과금 대상 입력으로 재생 | resume당 500K+ 토큰 |
| `/dream`, `/insights` | 백그라운드 API 호출이 보이지 않는 출력 없이 토큰 소모 | 무음 소모 |
| v2.1.89 이전 standalone | Sentinel 버그로 인해 지속적 4-17% cache read | 3-4배 토큰 낭비 |
| 자동 업데이트 활성화 | 향후 버전에서 리그레션이 재발할 수 있음 | 버그 3-5 수정 전까지 v2.1.91 고정 |

쉽게 말해, `--resume`은 이전 대화를 이어가는 기능인데, 대화 전체를 다시 보내기 때문에 토큰이 크게 소모됩니다. `/dream`이나 `/insights`는 사용자 눈에 보이지 않는 곳에서 토큰을 쓰는 명령어입니다.

## 주의해서 사용할 동작

완전히 피할 필요는 없지만, 알고 쓰면 좋은 것들입니다.

| 동작 | 이유 | 권장 사항 |
|------|------|----------|
| 병렬 서브 에이전트 (단일 터미널) | 각 에이전트가 새로운 컨텍스트로 시작하나 점차 안정화 | **안전** — 3-5회 요청 후 94-99%로 안정화 |
| 동시 다중 터미널 | 캐시 공유 없음, 병렬 쿼터 소모 | **활성 터미널을 하나로 제한** |
| 대용량 CLAUDE.md / 컨텍스트 파일 | 매 턴마다 전송됨 | 간결하게 유지 |
| 세션 시작 / compaction | `cache_creation` 스파이크는 구조적 현상 | 정상 — 이를 감안하여 예산 책정 |

"동시 다중 터미널"이란 Claude Code 창을 여러 개 동시에 띄워서 쓰는 것을 말합니다. 각 창이 캐시를 공유하지 않아서, 같은 양의 작업이라도 토큰이 두 배로 듭니다. 가능하면 한 번에 하나의 Claude Code 창만 쓰는 것이 좋습니다.

## 자가 진단

내 Claude Code가 영향을 받고 있는지 직접 확인하는 방법입니다.

### 세션 JSONL(세션 로그 파일) 확인

`~/.claude/projects/`에 있는 세션 파일에 매 턴(사용자와 AI가 한 번씩 주고받는 단위)의 사용량 데이터가 들어 있습니다.

- **정상 세션:** `cache_read` >> `cache_creation` (read 비율 > 80%) — 이전 대화 내용을 잘 재활용하고 있다는 뜻입니다
- **영향받는 세션:** `cache_creation` >> `cache_read` (read 비율 < 40%) — 매번 새로 보내고 있어서 토큰이 낭비되고 있다는 뜻입니다

대부분의 세션에서 낮은 read 비율이 나타나면, 문제가 있는 버전을 쓰고 있을 가능성이 높습니다. v2.1.91로 업데이트하십시오.

### 선택 사항: 프록시를 통한 캐시 모니터링

좀 더 정밀하게 확인하고 싶다면, 로컬 프록시(내 컴퓨터에서 돌리는 통신 기록 도구)를 사용할 수 있습니다. 이 방법은 기술적 경험이 있는 사용자에게 권장합니다.

```bash
# 모니터링을 위해 로컬 프록시를 경유
ANTHROPIC_BASE_URL=http://localhost:8080 claude

# 응답에서 cache_creation_input_tokens / cache_read_input_tokens 파싱
# 정상: read 비율 > 80%  |  영향받음: read 비율 < 40%
```

위 명령어를 터미널에 복사해서 붙여넣으면 됩니다. 단, 먼저 프록시 서버가 localhost:8080에서 실행 중이어야 합니다. 프록시 설정 방법은 이 레포지토리의 다른 문서를 참조하십시오.
